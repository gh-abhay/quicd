# LLM Context for superd

This file provides additional context for LLM agents working on the superd project.

## Project Goals

- Build a high-performance edge daemon for telecom services
- Achieve maximum throughput with non-blocking operations
- Use Sans-IO methodology for protocol implementations
- Support HTTP3/QUIC as the only transport protocol

## Key Technologies

- Rust programming language
- Tokio async runtime
- Quiche for QUIC implementation
- Sans-IO design pattern

## Development Environment

- Use rust-analyzer for IDE support
- Run `cargo check` frequently to validate changes
- Use `cargo clippy` for linting
- Format code with `cargo fmt`

## Workflow

1. Understand the Sans-IO pattern: separate logic from I/O
2. Implement services as state machines processing inputs to outputs
3. Test thoroughly, especially for concurrency and performance
4. Document public APIs and complex logic

## Performance Priorities

- Minimize allocations
- Avoid blocking operations
- Optimize hot paths
- Use appropriate data structures for high throughput
